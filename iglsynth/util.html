Utilities¶

Graph¶

A graph is defined as

\(G = \langle V, E, vprops, eprops, gprops \rangle\)

where

  • \(V\) is a set of vertices.
  • \(E\) is a set of edges.
  • \(vprops\) are vertex properties. Each vertex property maps the set of vertices to its property value.
  • \(eprops\) are edge properties. Each edge property maps the set of edges to its property value.
  • \(gprops\) are graph properties. A graph property is like a global variable for entire graph.

The properties \(vprops\), \(eprops\) and \(gprops\) can be one of the following types.

iglsynth.util.VALID_PROPERTY_TYPES (str) = {"bool", "int", "float", "string", "object"}¶

The API for Graph is as follows.

Sub-Graph¶

Given a graph (or sub-graph) \(G\), a sub-graph is a graph defined using boolean properties vfilt and/or efilt over \(G\). The vertices \(v \in V\) for which vfilt[v] = True are included in the sub-graph. Similarly, the edges \(e \in E\) for which efilt[e] = True are included in the sub-graph.

class iglsynth.util.SubGraph(graph: iglsynth.util.graph.Graph, vfilt_name: str = None, efilt_name: str = None)[source]¶

Represents a sub-graph of Graph defined using properties vfilt and/or efilt.

Parameters:
  • graph (Graph) – A graph object.
  • vfilt_name (str) – Name of the boolean vertex property used to define whether a vertex is in sub-graph or not.
  • efilt_name (str) – Name of the boolean edge property used to define whether an edge is in sub-graph or not.

Note

SubGraph is a derived class from Graph. All member functions and properties of Graph class apply to SubGraph.
