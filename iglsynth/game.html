Deterministic Game¶

A deterministic two-player game is defined as the graph

\({\cal G} = \langle V, E, vprops, eprops, gprops \rangle\)

where

  • \(V\) is a set of vertices.

  • \(E\) is a set of edges.

  • \(vprops\) are vertex properties.

    • Vertex properties must include is_final of type bool.
    • When \(\cal G\) is turn-based, vertex properties must include turn of type int.

  • \(eprops\) are edge properties.

    • Edge properties must include act of type int.

  • \(gprops\) are graph properties.

    • Graph property kind represents whether the game is Concurrent or Turn-based.

class iglsynth.game.game.Game(kind='Concurrent')[source]¶

Represents a deterministic two-player game. The game may be concurrent or turn-based.

Parameters:kind (str, either CONCURRENT or TURN_BASED) – Whether the game is concurrent or turn-based.
define(graph: iglsynth.util.graph.Graph = None, model: iglsynth.game.bases.Kripke = None, p1: iglsynth.game.bases.Player = None, p2: iglsynth.game.bases.Player = None, rp: Distribution = None, acc1: Acceptance = None, acc2: Acceptance = None)[source]¶

Define a two-player zero-sum game with given parameters. The instantiation checks for the following patterns, in order:

  1. game.define(graph=<Graph>)
  2. game.define(model=<TSys>, acc1=<Acceptance>)
  3. game.define(p1=<Player>, p2=<Player>, acc1=<Acceptance>)
Parameters:
  • graph (Graph) – Graph object representing game.
  • model (TSys) – A Kripke structure.
  • p1 (Player) – Player 1 profile.
  • p2 (Player) – Player 1 profile.
  • acc1 (Acceptance) – Winning condition of player 1.

Caution

Currently, only instantiation using graph is implemented.

graph¶

Returns the game graph.

kind¶

Returns the whether the game is turn-based or concurrent.
